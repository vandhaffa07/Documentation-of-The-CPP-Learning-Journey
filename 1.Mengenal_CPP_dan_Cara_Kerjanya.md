# MENGENAL C++ DAN CARA KERJANYA

## APA ITU C++?
C++ merupakan bahasa pemrograman serbaguna yang dikembangkan oleh Bjarne Stroustrup pada awal 1980-an. Sebagai evolusi dari bahasa C, C++ memperkenalkan paradigma Pemrograman Berorientasi Objek (OOP) yang meliputi class, inheritance, dan polymorphism sehingga memungkinkan pengembang membangun sistem kompleks secara lebih terstruktur. Meskipun telah ada selama beberapa dekade, C++ terus bertransformasi melalui pembaruan standar secara berkala. Hal ini membantah anggapan bahwa C++ adalah bahasa kuno, karena faktanya, fleksibilitas dan performanya yang luar biasa membuat C++ tetap menjadi fondasi utama dalam industri teknologi modern dan tetap menjadi pilihan utama para pakar IT di seluruh dunia.

## KARAKTERISTIK UTAMA C++
### 1. Compiled Language
C++ merupakan salah satu bahasa yang menggunakan mekanisme compile sebelum dapat dieksekusi. Artinya, source code yang ditulis di dalam file dengan ekstensi .cpp tidak bisa langsung dieksekusi oleh sistem, melainkan harus melalui serangkaian proses transformasi agar menjadi bahasa mesin terlebih dahulu. Serangkaian proses ini dilakukan oleh sebuah alat bernama compiler, dimana ia akan menerjemahkan seluruh syntax yang kita tulis menjadi instruksi yang dapat dipahami oleh CPU(biner atau assembly). Output akhir dari proses ini adalah file executable baru yang bersidat mandiri dan siap dijalankan. Karena proses kompilasi ini dilakukan sebelum program dijalankan, maka sebagaian besar kesalahan seperti syntax error atau kesalahan struktur program lainnya dapat terdeteksi lebih awal. Hal ini jelas berbeda dengan kebanyakan Interpreted Language(bahasa yang menggunakan mekanisme eksekusi baris per baris ketika program dijalankan, biasanya memerlukan interpreter tambahan) seperti python yang seringkali baru menampilkan error saat program sedang berjalan.

### 2. High Performance
C++ dikenal sebagai bahasa dengan performa tinggi karena hasil kompilasinya berupa kode mesin yang dieksekusi secara langsung oleh CPU. Tidak adanya layer tambahan seperti interprener atau virtual machine, menyebabkan instruksi yang ditulis dapat diterjemahkan menjadi operasi langsung yang sangat dekat dengan hardware. Hal ini membuat waktu eksekusi program menjadi lebih cepat dan penggunaan resource menjadi lebih efisien. Selain itu, C++ juga memberikan fleksibilitas kepada developer untuk mengoptimalkan program secara manual, baik dalam hal penggunaan memori maupun kontrol penuh atas komponen hardware yang ada, karena pada dasarnya, bahasa ini memang dirancang untuk memberikan kinerja maksimum dengan sedikit ketergantungan pada sistem operasi, sehingga membuatnya ideal untuk memanipulasi sistem tertanam dan driver perangkat.

### 3. Low Level Acces dan Manual Memory Management
Salah satu karakteristik penting C++ adalah kemampuannya untuk memberikan akses dan kontrol langsung terhadap memori. Melalui konsep seperti pointer, program dapat mengetahui dan memanipulasi alamat memori serta nilai yang disimpannya secara langsung. Melalui mekanisme seperti `new` developer dapat mengalokasikan memori secara manual untuk satu objek atau serangkaian objek pada saat runtime dan mengembalikan pointer ke memori yang baru dialokasikan, dan melalui mekanisme `delete` developer juga dapat membebaskan memori yang sebelumnya dialokasikan oleh operator new. Hal-hal tersebut memungkinkan kontrol yang sangat detail terhadap bagaimana data disimpan, dipindahkan, dan diubah di dalam sistem. Namun, berbeda dengan bahasa pemorograman lain yang memiliki garbage collector otomatis, C++ mengharuskan developer bertanggung jawab penuh terhadap pengelolaan memori. Jika memori yang dialokasikan tidak dibebaskan dengan benar, maka dapat terjadi memory leak, yaitu kondisi di mana memori terus terpakai tanpa bisa digunakan kembali. 

### 4. Multi Paradigm
C++ merupakan bahasa yang mendukung beberapa paradigma pemrograman yakni gaya, konsep, atau pendekatan mendasar dalam menyusun dan mengorganisir kode untuk menyelesaikan masalah komputasi, seperti Procedural programming (berbasis langkah-langkah instruksi), Object-Oriented Programming (berbasis objek dan class), dan Generic programming (menggunakan template untuk fleksibilitas tipe data). Kemampuan ini membuat C++ sangat fleksibel, karena dapat digunakan untuk berbagai jenis permasalahan dengan pendekatan yang berbeda. Sebagai contoh, program sederhana dapat ditulis secara prosedural, sementara sistem kompleks dapat menggunakan pendekatan object-oriented.

## FUNGSI DAN KEGUNAAN C++
### 1. Pengembangan Sistem Operasi
C++ sering digunakan dalam pengembangan sistem operasi seperti Windows, Linux, dan macOS karena kemampuannya dalam mengakses memori secara langsung serta berinteraksi dekat dengan hardware. Pada sistem operasi, komponen seperti kernel, manajemen memori, dan driver perangkat keras membutuhkan kontrol penuh terhadap resource yang ada di sistem. C++ menyediakan kemampuan tersebut melalui penggunaan pointer dan kontrol memori manual, yang memungkinkan developer menentukan secara langsung bagaimana data disimpan dan diakses di dalam memori. Berbeda dengan bahasa tingkat tinggi yang banyak melakukan abstraksi, C++ tetap memberikan akses low-level tanpa menghilangkan struktur pemrograman yang terorganisir.

### 2. Game Development
Dalam pengembangan game, C++ banyak digunakan terutama sebagai bagian dari game engine dan logika inti. Game modern, khususnya kategori AAA, membutuhkan performa real-time yang sangat tinggi, di mana setiap frame harus diproses dalam waktu yang sangat singkat. Keterlambatan kecil dalam proses ini dapat menyebabkan penurunan frame rate (lag) yang secara langsung mempengaruhi pengalaman pengguna. Oleh karena itu, dibutuhkan bahasa yang mampu mengeksekusi instruksi dengan sangat cepat dan efisien. C++ merupakan bahasa yang dikompilasi (compiled language), di mana seluruh kode sumber diterjemahkan menjadi kode mesin sebelum program dijalankan. Hal ini membuat eksekusi program menjadi jauh lebih cepat dibandingkan bahasa yang menggunakan interpreter atau virtual machine. 

### 3. Embedded System dan IOT
C++ merupakan salah satu bahasa yang sangat cocok untuk pengembangan embedded system dan Internet of Things (IoT) karena efisiensi dan kemampuannya dalam berinteraksi langsung dengan hardware. Pada sistem embedded, perangkat yang digunakan biasanya memiliki keterbatasan resource seperti memori kecil, daya rendah, dan prosesor dengan kemampuan terbatas. Oleh karena itu, diperlukan bahasa yang mampu menghasilkan kode dengan ukuran kecil dan performa tinggi seperti C++. C++ juga memungkinkan pengembang untuk mengakses perangkat keras secara langsung, seperti mikrokontroler (MCU), sensor, dan modul komunikasi, melalui manipulasi memori dan register. Hal ini memberikan kontrol penuh terhadap bagaimana perangkat tersebut bekerja.

## ALUR KERJA PROGRAM C++
### 1. Preprocessing
Tahap preprocessing pada proses kompilasi C++ adalah langkah pertama sebelum source code benar-benar diterjemahkan menjadi bahasa mesin. Pada fase ini, sebuah program khusus bernama preprocessor akan memindai kode dan mengeksekusi direktif-direktif khusus yang ditandai dengan simbol `#`. Preprocessor akan memperluas kode dengan menyisipkan isi sebenarnya dari file header yang telah kita sertakan (`#include <filename>`) dan mengganti makro dengan nilai atau ekspresi yang sesuai (`#define`). Hasil akhirnya adalah file perantara ber-ekstensi .i yang berisi kode sumber lengkap dan telah siap untuk masuk ke tahap selanjutnya.

Untuk memahami bagaimana preprocessing bekerja secara nyata, saya akan melakukan simulasi dengan menggunakan program C++ sederhana. Mula mula, saya akan membuat file C++ bernama file1.cpp menggunakan perintah cat seperti ini :
```bash
cat > halo1.cpp
#include <iostream>
using namespace std;

int main(){
cout<<"Halo teman-teman!"<<endl;
cin.get();
return 0;
}
```
Secara kasat mata, program ini terlihat sangat sederhana dan hanya terdiri dari beberapa baris kode. Program ini hanya melakukan output ke layar dan menunggu input enter sebelum akhirnya benar-benar selesai. **(Catatan : Syntax-syntax code tersebut akan dibahas lebih lanjut pada dokumentasi pembelajaran selanjutnya)**

Selanjutnya, untuk melihat apa yang sebenarnya terjadi pada tahap preprocessing, gunakan perintah seperti ini:
```bash
g++ -E halo1.cpp -o halo1.i
```
Perintah ini menjalankan compiler C++, tetapi dengan opsi khusus -E yang menghentikan proses hanya sampai pada tahap preprocessing saja. Artinya, compiler tidak akan mengubah kode menjadi object file, melakukan linking, ataupun menghasilkan file executable, melainkan membuat file perantara ber-ekstensi .i yang berisi source code secara lengkap setelah seluruh directive seperti #include dan macro diproses sepenuhnya.

Kemudian, saya akan membuka file tersebut menggunakan teks editor nano dengan option -l untuk melihat seberapa banyak baris kode pada source code ketika telah melalui proses preprocessing :
```bash
nano -l halo.i
```

Hasil akhirnya adalah satu file besar yang berisi seluruh kode yang dibutuhkan untuk dikompilasi, tanpa ketergantungan eksternal dalam bentuk include.

### 2. Compiling
Tahap compiling merupakan proses lanjutan setelah preprocessing, di mana source code yang telah diperluas akan diterjemahkan menjadi bentuk yang lebih dekat dengan bahasa mesin. Pada fase ini, compiler akan melakukan beberapa proses penting, seperti memeriksa apakah kode yang ditulis sesuai dengan aturan bahasa C++. Kesalahan seperti kurang tanda titik koma (;) atau kesalahan penulisan keyword akan terdeteksi pada tahap ini. Compiler juga akan memastikan bahwa kode memiliki makna yang valid. Misalnya, penggunaan variabel yang belum dideklarasikan atau operasi yang tidak sesuai dengan tipe data akan dianggap sebagai error. Setelah kode dinyatakan valid, compiler akan menerjemahkan kode tersebut menjadi object file, yaitu representasi kode dalam bentuk yang lebih dekat ke bahasa mesin, tetapi belum sepenuhnya siap dijalankan.

Untuk melihat hasil dari tahap compiling, kita dapat menggunakan perintah seperti ini:
```bash
g++ -c halo1.cpp -o halo1.o
```
Perintah ini akan menjalankan compiler dengan opsi -c, yang berarti proses hanya berhenti pada tahap compiling tanpa melanjutkan ke tahap linking. Output yang dihasilkan adalah file halo1.o yang disebut sebagai object file.

### 3. Linking
Tahap linking merupakan proses lanjutan setelah compiling, di mana seluruh object file akan digabungkan dengan library yang dibutuhkan untuk membentuk sebuah program yang utuh dan dapat dijalankan. Untuk memahami tahap linking, penting untuk membedakan antara deklarasi dan implementasi. Saat menggunakan directive `#include <iostream>`, preprocessor hanya menyisipkan deklarasi yang memberi tahu compiler bahwa terdapat objek-objek seperti `cout`, `cin`, dan `endl` pada kode program, tetapi tidak menjelaskan bagaimana objek tersebut bekerja secara internal. Pada tahap compiling, ketika compiler menemukan kode seperti `cout << "Halo teman-teman!" << endl;`, compiler akan menganggap kode tersebut valid karena `cout` dan `endl` telah dideklarasikan sebelumnya melalui `#include`. Compiler juga akan menyimpan referensi terhadap simbol-simbol lain seperti `cout`, `cin`, dan `endl`, tetapi tetap tidak mengeksekusi atau menyertakan implementasi dari simbol-simbol tersebut. Pada tahap linking barulah dilakukan penyelesaiaan untuk seluruh referensi simbol yang belum memiliki implementasi. Linker akan membaca seluruh simbol yang terdapat dalam object file, mencari implementasi simbol tersebut di dalam standard library C++ (seperti libstdc++), menghubungkan referensi simbol dengan implementasi aslinya, dan menentukan alamat memori final untuk setiap fungsi dan objek. Setelah semua simbol berhasil diselesaikan, linker akan menghasilkan file executable mandiri yang siap untuk dijalankan. 

Untuk melakukan linking, gunakan perintah seperti ini :
```bash
 g++ halo1.o -o halo1
```
Perintah ini akan menjalankan proses linking terhadap object file halo1.o dan menghasilkan file executable bernama halo1. Setelah proses linking selesai, program dapat dijalankan dengan perintah:
```bash
./halo1
Halo teman-teman!

```
Dapat terlihat bahwa program berjalan sesuai dengan kode yang saya tulis sebelumnya. Hal ini menunjukkan bahwa seluruh referensi simbol yang sebelumnya belum memiliki implementasi telah berhasil dihubungkan dengan implementasi sebenarnya dari standard library C++.
